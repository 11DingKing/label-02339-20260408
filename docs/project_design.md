# 环境监测系统 - 项目设计文档

## 1. 项目概述

### 1.1 项目背景

本系统是一款基于 Qt 6 开发的环境参数实时监测与管理系统，用于采集、显示、存储和上传环境数据（温度、湿度、光照强度）。

### 1.2 功能需求

| 模块 | 功能描述 |
|------|----------|
| UI界面 | 主界面布局、实时数据展示、历史数据查询、参数设置 |
| 串口通信 | 串口参数配置、数据收发、模拟模式、状态监测 |
| 网络通信 | TCP/UDP协议、数据上传、自动重连、状态监测 |
| 数据库 | SQLite存储、增删查操作、异常处理 |
| 配置管理 | 配置文件读写、参数持久化 |

### 1.3 技术选型

| 技术 | 选择 | 说明 |
|------|------|------|
| 开发框架 | Qt 6.x | 跨平台GUI框架 |
| 编程语言 | C++17 | 现代C++标准 |
| 构建工具 | CMake 3.16+ | 跨平台构建 |
| 数据库 | SQLite | 轻量级嵌入式数据库 |
| 串口库 | Qt SerialPort | Qt官方串口模块 |
| 网络库 | Qt Network | Qt官方网络模块 |

## 3. 模块设计

### 3.1 主窗口模块 (MainWindow)

**职责**：UI布局、控件绑定、用户交互、信号槽连接

**主要方法**：

| 方法 | 功能 | 说明 |
|------|------|------|
| setupUI() | 初始化界面 | 创建布局、控件、样式 |
| setupMenuBar() | 创建菜单栏 | 文件、设置、帮助菜单 |
| setupToolBar() | 创建工具栏 | 设置、一键模拟、查询历史 |
| setupStatusBar() | 创建状态栏 | 串口、网络、数据库状态 |
| onDataUpdated() | 数据更新槽 | 更新仪表盘和数值显示 |
| onQuickSimulate() | 一键模拟 | 启动串口模拟+网络连接 |
| onQueryHistory() | 查询历史 | 查询并显示历史数据 |

### 3.2 设置对话框模块 (SettingsDialog)

**职责**：串口/网络/采集参数配置

**主要功能**：

| 标签页 | 功能 |
|--------|------|
| 串口设置 | 串口选择、参数配置、连接/断开、模拟模式、数据发送 |
| 网络设置 | 协议选择、服务器配置、连接/断开、自动上传、手动上传 |
| 采集设置 | 采集周期配置 |

### 3.3 串口管理模块 (SerialManager)

**职责**：串口通信、数据收发、模拟数据生成

**主要方法**：

| 方法 | 功能 |
|------|------|
| openPort() | 打开串口，配置参数 |
| closePort() | 关闭串口 |
| sendData() | 发送数据 |
| startSimulation() | 启动模拟模式 |
| stopSimulation() | 停止模拟模式 |

**信号**：

| 信号 | 触发时机 |
|------|----------|
| dataReceived(SensorData) | 接收到有效数据 |
| connectionStatusChanged(bool) | 连接状态变化 |
| errorOccurred(QString) | 发生错误 |
| portDisconnected() | 串口意外断开 |

### 3.4 网络管理模块 (NetworkManager)

**职责**：TCP/UDP通信、数据上传、自动重连

**主要方法**：

| 方法 | 功能 |
|------|------|
| connectToServer() | 连接服务器 |
| disconnectFromServer() | 断开连接 |
| uploadData() | 上传数据 |
| manualUpload() | 手动触发上传 |
| setAutoUpload() | 设置自动上传 |
| resetReconnect() | 重置重连计数器 |

**重连策略**：
- 指数退避：3秒 → 6秒 → 12秒 → 24秒 → ... → 最大60秒
- 无次数上限，持续重连直到成功

### 3.5 数据库管理模块 (DatabaseManager)

**职责**：SQLite数据库操作

**主要方法**：

| 方法 | 功能 |
|------|------|
| initialize() | 初始化数据库，创建表 |
| insertRecord() | 插入数据记录 |
| queryByTimeRange() | 按时间范围查询 |
| deleteByTimeRange() | 按时间范围删除 |
| getRecordCount() | 获取记录总数 |

### 3.6 数据处理模块 (DataProcessor)

**职责**：数据分发、存储协调、上传协调

**工作流程**：
1. 接收 SerialManager 的原始数据
2. 解析并验证数据
3. 存储到数据库
4. 发送到网络模块上传
5. 通知UI更新显示

### 3.7 工具类模块

**StringUtils**：
- isValidIpAddress() - IP地址验证
- isValidPort() - 端口号验证
- formatSensorData() - 数据格式化

**LogManager**：
- initialize() - 初始化日志系统
- info/warning/error() - 记录不同级别日志
- 自动按日期分割日志文件

## 4. 数据库设计

### 4.1 ER 图

```
┌─────────────────────────────────────┐
│          environment_data           │
├─────────────────────────────────────┤
│ id            INTEGER  PK 自增主键  │
│ collect_time  TEXT     采集时间     │
│ temperature   REAL     温度(℃)     │
│ humidity      REAL     湿度(%)      │
│ light_intensity REAL   光照(Lux)    │
└─────────────────────────────────────┘
```

### 4.2 建表语句

```sql
CREATE TABLE IF NOT EXISTS environment_data (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    collect_time TEXT NOT NULL,
    temperature REAL NOT NULL,
    humidity REAL NOT NULL,
    light_intensity REAL NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_collect_time 
ON environment_data(collect_time);
```

## 7. 错误处理设计

### 7.1 错误分类

| 类别 | 示例 | 处理方式 |
|------|------|----------|
| 串口错误 | 设备断开、权限不足 | 提示用户检查设备连接 |
| 网络错误 | 连接失败、超时 | 自动重连，提示用户检查网络 |
| 数据库错误 | 写入失败、查询失败 | 记录日志，提示用户 |
| 数据错误 | 格式错误、范围越界 | 忽略无效数据，记录日志 |

### 7.2 错误提示规范

错误提示应包含：
1. 错误描述（发生了什么）
2. 可能原因（为什么发生）
3. 解决建议（如何解决）

示例：
```
错误：串口连接失败
原因：设备可能未连接或被其他程序占用
建议：请检查设备连接，关闭其他串口程序后重试
```
