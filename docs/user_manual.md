## 系统概述

环境监测系统是一款基于 Qt 6 开发的桌面应用程序，用于实时监测和记录环境参数（温度、湿度、光照强度）。

### 主要功能

- **实时数据监测**：通过仪表盘和数值实时显示环境参数
- **串口通信**：支持连接真实传感器设备或使用模拟模式
- **网络上传**：支持 TCP/UDP 协议将数据上传至远程服务器
- **历史数据管理**：使用 SQLite 数据库存储和查询历史记录
- **配置管理**：支持保存和加载配置文件

### 系统要求

| 项目 | 最低要求 |
|------|----------|
| 操作系统 | macOS 12+ / Ubuntu 22.04+ / Windows 10+ |
| 内存 | 4GB |
| 磁盘空间 | 500MB |
| 显示器 | 1280×800 分辨率 |

## 安装与启动

### 方式一：一键启动（推荐）

**macOS / Linux：**
```bash
# 1. 打开终端，进入项目目录
cd /path/to/EnvMonitor

# 2. 添加执行权限
chmod +x run.sh

# 3. 运行启动脚本
./run.sh
```

**Windows：**
```cmd
# 双击运行 run.bat
# 或在命令提示符中执行：
run.bat
```

脚本会自动完成以下操作：
1. 检测并安装依赖（Qt6、CMake）
2. 编译项目
3. 初始化数据库
4. 启动测试服务器（用于网络功能测试）
5. 启动程序

### 方式二：手动编译

详见 README.md 中的手动编译说明。

## 功能操作指南

### 一、快速体验（一键模拟）

这是最快速体验系统全部功能的方式：

1. 启动程序后，点击工具栏「一键模拟」按钮
2. 系统将自动：
   - 启动串口模拟，生成随机环境数据
   - 连接测试服务器（127.0.0.1:9000）
   - 开始自动上传数据
3. 观察：
   - 仪表盘数值实时变化
   - 日志区显示数据接收和上传记录
   - 状态栏显示连接状态
4. 再次点击「一键模拟」停止

### 二、串口功能操作

#### 2.1 连接真实串口设备

1. 将传感器设备通过串口线连接到电脑
2. 点击「设置」打开设置对话框
3. 在「串口设置」标签页：
   - 点击「刷新」获取可用串口列表
   - 选择对应的串口（如 COM3 或 /dev/ttyUSB0）
   - 配置参数（波特率、数据位、校验位、停止位、流控）
   - 点击「连接」
4. 连接成功后，状态栏显示「串口: 已连接」

#### 2.2 使用模拟模式

无需真实设备，可使用模拟模式测试：

1. 点击「设置」→「串口设置」
2. 点击「模拟」按钮
3. 系统开始生成模拟数据
4. 按钮变为「停止模拟」，再次点击可停止

#### 2.3 发送串口数据

1. 确保串口已连接
2. 在「数据发送」区域输入要发送的内容
3. 如需发送十六进制数据，勾选「HEX模式」
4. 点击「发送」按钮

### 三、网络功能操作

#### 3.1 连接服务器

1. 点击「设置」→「网络设置」
2. 配置服务器参数：
   - 协议：TCP 或 UDP
   - 地址：服务器 IP 地址（测试用 127.0.0.1）
   - 端口：服务器端口（测试用 9000）
3. 点击「连接」按钮
4. 连接成功后，状态栏显示「网络: 已连接」

#### 3.2 数据上传

**自动上传：**
1. 勾选「自动」复选框
2. 设置上传间隔（毫秒）
3. 系统将按设定间隔自动上传数据

**手动上传：**
1. 点击「手动上传数据」按钮
2. 系统立即上传当前数据

#### 3.3 断线重连

- 网络断开后，系统自动尝试重连
- 重连间隔逐渐增加：3秒 → 6秒 → 12秒 → ... → 最大60秒
- 可点击「重连」按钮手动触发重连

### 四、历史数据操作

#### 4.1 查询历史数据

1. 在「历史数据查询」区域设置时间范围：
   - 起始时间：选择查询开始时间
   - 结束时间：选择查询结束时间
2. 点击「查询」按钮
3. 表格显示查询结果，右上角显示记录数

#### 4.2 删除历史数据

1. 设置要删除的时间范围
2. 点击「删除」按钮
3. 弹出确认对话框，点击「确定」执行删除
4. 删除后自动刷新表格

### 五、配置管理

#### 5.1 保存配置

1. 菜单 → 文件 → 保存配置
2. 当前所有设置将保存到配置文件

#### 5.2 加载配置

1. 菜单 → 文件 → 加载配置
2. 选择 .ini 配置文件
3. 系统加载配置并更新界面

## 常见问题

### Q1: 启动时提示缺少 Qt 库

**解决方案：**
- macOS：运行 `brew install qt@6`
- Ubuntu：运行 `sudo apt install qt6-base-dev`
- Windows：下载安装 Qt 6.x

### Q2: 串口列表为空

**解决方案：**
1. 确认设备已正确连接
2. 检查设备驱动是否安装
3. 点击「刷新」按钮重新获取列表
4. macOS/Linux 用户检查串口权限

### Q3: 网络连接失败

**解决方案：**
1. 确认服务器地址和端口正确
2. 检查防火墙设置
3. 确认服务器正在运行
4. 测试时使用一键启动脚本（自动启动测试服务器）

### Q4: 数据库错误

**解决方案：**
1. 检查磁盘空间是否充足
2. 确认程序有写入权限
3. 删除 envmonitor.db 文件后重新启动（数据会丢失）

### Q5: 仪表盘不显示数据

**解决方案：**
1. 确认串口已连接或模拟模式已启动
2. 检查日志区是否有数据接收记录
3. 尝试点击「一键模拟」快速测试
